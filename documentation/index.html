What Does It Do?

This library is an add-on for SuperScript. It is strongly recommended to read the documentation for the core project before proceeding with this library. If you haven't yet read the core documentation then it is recommended that you start by reading about the core concepts.

SuperScript.Templates.jQueryTmpl adds functionality for declaring HTML templates, written for the JavaScript Templates library in server-side .NET code.

While the SuperScript library contains the core functionality for making and emitting declarations, SuperScript.Templates.jQueryTmpl contains a specific implementation of one of those core classes. Broadly speaking, SuperScript has three main aspects: declarations, which are instances of DeclarationBase, and emitters, which consist of a combination of modifiers.

SuperScript.Templates.jQueryTmpl therefore consists an implementation of a specific modifier, an CollectionConverter. In fact, this implementation, jQueryTmplStringify, is part of a hierarchical chain:

  • jQueryTmplStringify
  • ↓
  • SuperScript.Templates.Modifiers.Converters.TemplateStringify
  • ↓
  • SuperScript.Modifiers.Converters.CollectionConverter
  • ↓
  • SuperScript.Modifiers.ModifierBase

The reason that the implementations of TemplateStringify differ is because of the PreCompile option. Naturally, the compilation action for each type of HTML client library differs (otherwise they would all be the same library).

Therefore, the only real difference between implementations of TemplateStringify lies in the implementations of CollateTemplates method.

Consequently, this implementation offers the ability to pre-compile JavaScript Templates HTML templates whereas other implementations of TemplateStringify will offer compilation into different client-side template library formats.

Example

Here is an example of declaratively configuring an IEmitter in a config file. This emitter implements this jQueryTmplStringify class and uses TemplateStringifyJavaScript to write out the contents.

This example also shows how to enable pre-compilation. 

	<configuration>
		<configSections>
			<section name="superScript" type="SuperScript.Configuration.SuperScriptConfig, SuperScript.Common" />
		</configSections>
		…
		<superScript>
			<emitter key="templates">
				<converters>
					<modifier type="SuperScript.Templates.jQueryTmpl.Modifiers.Converters.jQueryTmplStringify, SuperScript.Templates.jQueryTmpl">
						<properties>
							<property name="PreCompile" value="true" type="System.Boolean, mscorlib" />
						</properties>
					</modifier>
				</converters>
				<writers>
					<modifier type="SuperScript.Templates.Modifiers.Writers.TemplateStringifyJavaScript, SuperScript.Templates" />
				</writers>
			</emitter>
		</superScript>
	</configuration>

The same emitter can also be constructed programmatically. 

	using SuperScript.Configuration;
	…
	Settings.Instance.Emitters.Add(new StandardEmitter
                                         {
                                             Converter = new SuperScript.Templates.jQueryTmpl.Modifiers.Converters.jQueryTmplStringify
                                                             {
                                                                 PreCompile = true
                                                             },
                                             Key = "templates",
                                             HtmlWriter = new SuperScript.Templates.Modifiers.Writers.TemplateStringifyJavaScript()
                                         });
